# Dev checklist (v4) — uDosDev

Short pass before/after governance or script changes.

## Pre

- [ ] Change is scoped (docs vs scripts vs automation)
- [ ] Related entry exists or will exist in **`TASKS.md`**

## Post

- [ ] Scripts still run if touched (or documented manual step)
- [ ] No secrets or inbox content committed
- [ ] Task moved to **Done** when finished

## Rule

Governance repos drift when work lives only in chat. If it matters, it belongs in **`TASKS.md`** or **`docs/`**.
