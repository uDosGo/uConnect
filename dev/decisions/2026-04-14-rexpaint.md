# Decision: REXPaint Bridge in A1 Core

- Date: 2026-04-14
- Status: locked
- Scope: `udo grid import`, `udo grid export`, `core-rs/src/rexpaint/*`

## Decision

Use `.xp` (REXPaint) as an import/export bridge format for grid workflows in A1 core, with OBF as the internal text-friendly representation.

## Why

- REXPaint gives creators a mature external grid authoring tool.
- OBF keeps project surfaces versionable and portable inside uDos docs/vault workflows.

## Acceptance Criteria

- [x] `udo grid import map.xp --out map.obf` writes OBF output
- [x] `udo grid export map.obf --format xp --out map.xp` writes XP output
- [x] CLI integration tests cover import/export file outputs

## Notes

- A1 supports `xp` format for export.
- Additional grid formats can be layered in later milestones.
