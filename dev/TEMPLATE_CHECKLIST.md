# DEV_CHECKLIST.md — [Project/Milestone]

Use this before and after each dev pass.

## Pre-pass (30-60s)

- [ ] I understand current context (`README`, `dev`, `TASKS.md`)
- [ ] I am working on one clear task
- [ ] Scope is explicit (what is included and excluded)
- [ ] File placement is clear before editing

## During pass

- [ ] Stay in scope (no silent side quests)
- [ ] Keep logs concise and useful
- [ ] Keep debug traces temporary
- [ ] Error messages remain actionable

## Post-pass (before commit)

- [ ] Main path works
- [ ] Basic error path verified
- [ ] No leftover debug noise
- [ ] No dead code/scratch in active folders
- [ ] Correct placement (`/src`, `/docs`, `/dev`, `.local`, `.compost`)

## Task hygiene

- [ ] Task state updated in `TASKS.md`
- [ ] Blocked tasks include reason
- [ ] No hidden work outside tracked tasks

## Round mode check

- [ ] If using rounds, continue immediately to next round on deliverable completion
- [ ] Pause only if explicit checkpoint or high-risk blocker

## Commit quality

- [ ] One clear change per commit
- [ ] No unrelated files bundled
- [ ] Commit message explains intent
