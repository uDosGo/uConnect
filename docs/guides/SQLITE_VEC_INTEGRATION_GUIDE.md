# SQLite-vec Integration Guide

This guide provides step-by-step instructions for migrating from mock vector search to real sqlite-vec implementation.

## Current State

The system currently runs in mock mode with the following configuration:

```rust
// core-rs/src/mcp/vector/mod.rs
pub const VECTOR_SEARCH_ENABLED: bool = false; // Default to disabled until sqlite-vec is available
```

## Prerequisites

### 1. Install sqlite-vec Extension

You need to install the sqlite-vec extension. There are two options:

**Option A: Pre-compiled Binary**
```bash
# Download pre-compiled sqlite-vec extension for your platform
# Check https://github.com/asg017/sqlite-vec/releases for latest version
wget https://github.com/asg017/sqlite-vec/releases/download/v0.1.0/sqlite-vec-linux-amd64.so
```

**Option B: Compile from Source**
```bash
# Clone and build sqlite-vec
git clone https://github.com/asg017/sqlite-vec.git
cd sqlite-vec
make
# This will produce vec0.so extension file
```

### 2. Verify SQLite Version

Ensure your SQLite version is compatible with sqlite-vec:

```bash
sqlite3 --version
# Should be SQLite 3.35.0 or higher
```

## Integration Steps

### Step 1: Enable Vector Search Feature

```rust
// core-rs/src/mcp/vector/mod.rs
pub const VECTOR_SEARCH_ENABLED: bool = true; // Enable real vector search
```

### Step 2: Load sqlite-vec Extension

Add the extension loading code to your database initialization:

```rust
// In your database connection setup
use rusqlite::Connection;

fn load_vector_extension(conn: &Connection) -> Result<(), Box<dyn std::error::Error>> {
    // Path to sqlite-vec extension
    let extension_path = "path/to/vec0.so"; // or vec0.dll on Windows
    
    // Enable extension loading
    conn.execute("SELECT load_extension(?1)", [extension_path])?;
    
    // Verify extension loaded
    let mut stmt = conn.prepare("SELECT vec_version()")?;
    let version: String = stmt.query_row([], |row| row.get(0))?;
    println!("sqlite-vec version: {}", version);
    
    Ok(())
}
```

### Step 3: Create Vector Index Tables

Run the database migration to create vector index tables:

```bash
# Apply the vector migration
udo db migrate vector
```

This will execute the SQL from `core-rs/src/mcp/vector/schema.rs`:

```sql
-- Vector index table for feed items
CREATE VIRTUAL TABLE IF NOT EXISTS feed_vector_index USING vec0(
    content TEXT,
    embedding float[1536]  -- Adjust dimension based on your embedding model
);

-- Create index for fast similarity search
CREATE INDEX IF NOT EXISTS idx_feed_vector ON feed_vector_index(embedding);
```

### Step 4: Generate Real Embeddings

Use the vectorization command to generate real embeddings:

```bash
# Generate embeddings for all feed items
udo feed vector vectorize --all

# Or for specific feeds
udo feed vector vectorize --feed-id FEED_ID
```

This will replace the mock embeddings with real ones generated by your embedding model.

### Step 5: Test Vector Search

Verify that vector search is working with real embeddings:

```bash
# Test vector search
udo feed vector search --query "test query" --limit 10

# Test hybrid search (vector + keyword)
udo feed vector hybrid --query "test query" --limit 10
```

## Configuration Options

### Embedding Model Configuration

Configure your embedding model in the MCP configuration:

```toml
# core-rs/config.toml
[vector_search]
enabled = true
embedding_model = "all-MiniLM-L6-v2"  # or your preferred model
embedding_dimension = 384
max_tokens = 512
batch_size = 32
```

### Performance Tuning

Optimize sqlite-vec performance with these parameters:

```rust
// In your vector search implementation
let search_options = VectorSearchOptions {
    top_k: 10,           // Number of results to return
    ef_search: 128,      // Search depth (higher = more accurate but slower)
    ef_construction: 64, // Index construction parameter
    max_distance: 0.8,   // Maximum cosine distance for results
};
```

## Troubleshooting

### Common Issues

**Extension Loading Failed**
- Verify the extension path is correct
- Check file permissions
- Ensure SQLite was compiled with extension support

**Vector Search Returns No Results**
- Verify embeddings were generated (`SELECT COUNT(*) FROM feed_vector_index`)
- Check your distance threshold isn't too strict
- Verify the query produces valid embeddings

**Performance Issues**
- Increase `ef_search` for better accuracy (slower)
- Decrease `ef_search` for faster queries (less accurate)
- Consider reducing embedding dimension if response is too slow

## Rolling Back

If you need to revert to mock mode:

```rust
// core-rs/src/mcp/vector/mod.rs
pub const VECTOR_SEARCH_ENABLED: bool = false; // Revert to mock mode
```

The system will automatically fall back to mock implementations.
