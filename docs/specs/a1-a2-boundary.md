# A1/A2 boundary matrix

## Core rule

- **GitHub = A1-native**
- **WordPress/cloud orchestrator = A2-only**
- A1 stubs are contracts that return upgrade guidance.

## A1-native

- GitHub issues/PR workflow
- GitHub Pages deploy
- Local vault operations
- Markdown, OBF grid, USXD tooling
- Local publish preview

## A2-only (stubbed in A1)

- WordPress sync/publish/review
- Orchestrator webhooks
- OAuth2 provider
- Realtime cloud events
- Multi-tenant cloud server
- Multi-device cloud sync

## Precedence

1. Commercial/cloud boundary
2. A1-safe local/GitHub features
3. Explicit exceptions listed in `LOCKED-REGISTRY.md`
