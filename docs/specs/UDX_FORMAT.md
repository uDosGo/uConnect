# UDX Format Specification
# Universal Document eXchange - Code Atlas Format

**Version:** 1.0
**Status:** Final
**Last Updated:** 2024-04-24

## 📋 Overview

UDX (Universal Document eXchange) is a JSON-based format for organizing code, documentation, and assets in a unified code atlas structure. It's designed for uDos's code management system and provides a structured way to map documents to code entities.

## 🗂️ Format Structure

### Root Object

```json
{
  "version": "1.0",
  "type": "layout",
  "title": "Document Title",
  "description": "Document description",
  "sections": [...],
  "metadata": {...},
  "codeAtlas": {...}
}
```

### Root Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `version` | string | Yes | Format version (currently "1.0") |
| `type` | string | Yes | Document type (e.g., "layout", "api", "component") |
| `title` | string | Yes | Document title |
| `description` | string | No | Document description |
| `sections` | array | Yes | Array of document sections |
| `metadata` | object | Yes | Document metadata |
| `codeAtlas` | object | No | Code atlas mappings |

## 📖 Section Types

### Base Section Structure

```json
{
  "id": "unique-identifier",
  "type": "section-type",
  "title": "Section Title",
  "content": {...}
}
```

### Supported Section Types

#### 1. Header Section

```json
{
  "id": "header",
  "type": "header",
  "title": "Document Title",
  "subtitle": "Document Subtitle",
  "metadata": {
    "author": "Author Name",
    "created": "2024-04-24",
    "version": "1.0.0"
  }
}
```

**Properties:**
- `title`: Main title
- `subtitle`: Optional subtitle
- `metadata`: Author and version info

#### 2. Content Section

```json
{
  "id": "main_content",
  "type": "content",
  "blocks": [
    {
      "id": "intro",
      "type": "markdown",
      "content": "# Introduction\n\nContent here",
      "style": "prose"
    }
  ]
}
```

**Block Types:**
- `markdown`: Markdown content
- `list`: Bullet or numbered list
- `code`: Code block with syntax highlighting
- `table`: Data table
- `image`: Image reference

#### 3. Footer Section

```json
{
  "id": "footer",
  "type": "footer",
  "content": "Generated by uDos",
  "links": [
    {"text": "Docs", "url": "https://example.com"}
  ]
}
```

**Properties:**
- `content`: Footer text
- `links`: Array of link objects

## 🧭 Code Atlas System

### Code Atlas Structure

```json
{
  "version": "1.0",
  "mappings": {
    "section-id": "code-reference",
    "block-id": "code:module:entity"
  }
}
```

### Mapping Format

**Code Reference Types:**
- `doc:*` - Documentation reference
- `code:*` - Code entity reference
- `api:*` - API endpoint reference
- `test:*` - Test reference

**Examples:**
- `doc:intro` - Documentation introduction
- `code:parser:rust` - Rust parser module
- `api:v1:users` - Users API endpoint
- `test:unit:core` - Core unit tests

```json
    }
  }

```

## 🔧 Validation Rules

1. **Required Fields**: All required fields must be present
2. **Unique IDs**: Section and block IDs must be unique
3. **Valid Types**: Section types must be from supported list
4. **Valid JSON**: Document must be valid JSON
5. **Code References**: Atlas mappings must use valid reference format

## 📚 Usage in uDos

### File Extension
- `.udx` - Universal Document eXchange files

### Storage Location
- `~/Code/Vault/.udx/` - UDX document storage

### Integration
- Loaded by uCode1's vault-bridge
- Parsed by code atlas system
- Mapped to code entities via MCP

## 🎯 Code Atlas Features

### Entity Mapping
- Maps document sections to code entities
- Enables bidirectional navigation
- Supports version tracking
- Provides impact analysis

### Reference Types
- **Documentation → Code**: `doc:*` → `code:*`
- **Code → Documentation**: `code:*` → `doc:*`
- **API → Implementation**: `api:*` → `code:*`
- **Tests → Code**: `test:*` → `code:*`
