# Legacy — Outdated / Superseded Documentation

This directory holds documentation that is **clearly outdated or superseded** — content that:

- Has been replaced by a newer version of the same doc
- Describes an implementation that no longer exists
- Was a proposal/spec that was never adopted and is now stale
- Is a duplicate that has been consolidated elsewhere

## Purpose

We don't delete docs — we archive them. Legacy docs preserve:

- **Historical context** — why decisions were made
- **Great ideas** that may not have been fully implemented yet
- **Specs that morphed** into something different
- **Institutional knowledge** — what was tried and what didn't work

## What Goes Here

- Superseded architecture docs (e.g., old ARCHITECTURE.md replaced by CORE_ARCHITECTURE.md)
- Completed phase roadmaps (replaced by consolidated roadmaps)
- Duplicate guides that now live canonically elsewhere
- Experimental specs that never shipped
- Implementation notes for systems that have been rewritten

## What Does NOT Go Here

- Active, current documentation (stays in `docs/`)
- Work-in-progress or uncertain docs (goes in `../review/`)
- Trivial or personal notes (should be deleted or moved to personal vault)
