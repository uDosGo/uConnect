# Decision: Rust Core for A1 Backend

- Date: 2026-04-14
- Status: locked
- Scope: `core-rs/`, CLI backend runtime, local-first A1 execution

## Decision

Adopt Rust as the backend/runtime language for uDos A1 core and treat `core-rs/` as the active implementation lane. The existing TypeScript core remains as legacy/reference during transition and is not the forward backend target.

## Why

- Local-first A1 runtime needs a small, fast binary with strong typing and predictable memory behavior.
- Rust aligns with terminal-first USXD rendering and native file-system operations.
- This keeps TypeScript focused on UI/editor surfaces (Vue + Milkdown), not backend runtime.

## Acceptance Criteria

- [x] `core-rs/` crate exists and builds (`cargo check`)
- [x] P0 vault commands are available (`init`, `list`, `delete`, `restore`, `search`)
- [x] CLI integration tests exist in `core-rs/tests/`

## Notes

- A1 remains local-first.
- A2 cloud workflows continue as stubs/contracts only in this lane.
