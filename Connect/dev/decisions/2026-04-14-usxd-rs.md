# Decision: USXD-RS Terminal Renderer

- Date: 2026-04-14
- Status: locked
- Scope: `core-rs/src/usxd/*`, `udo usxd render`

## Decision

Implement USXD rendering in Rust for A1 terminal usage, with mode-based output (`teletext`, `mono`, `wireframe`) and JSON/YAML parsing for surface files.

## Why

- USXD is a core operator-facing format in A1 and requires direct CLI rendering parity.
- Rust renderer path enables cohesive integration with vault and grid tooling in one binary.

## Acceptance Criteria

- [x] `udo usxd render <file> --mode teletext|mono|wireframe` works
- [x] Parser supports JSON and YAML surface files
- [x] Mode routing is covered by integration tests

## Notes

- Current render path is terminal output-first.
- Ratatui renderer foundation is in place for richer interactive modes.
