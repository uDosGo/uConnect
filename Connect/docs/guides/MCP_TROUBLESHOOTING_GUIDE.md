# MCP Troubleshooting Guide: "No MCP custom connections found"

## Issue
Le Chat reports: **"No MCP custom connections found"** when trying to use custom MCP tools.

## Root Cause
This error occurs when:
1. No MCP servers are registered with Le Chat
2. The registered MCP server is unreachable
3. The MCP manifest is invalid or missing
4. Authentication is failing

## Solution Steps

### 1. Verify MCP Server is Running

Check if your MCP server is running locally:
```bash
# Check if server is running
ps aux | grep node

# If not running, start it
node server.js

# Check logs
tail -f server.log
```

### 2. Expose Server with ngrok (for local testing)

```bash
# Install ngrok (if not installed)
brew install ngrok/ngrok/ngrok

# Expose local server
ngrok http 3000

# Note the HTTPS URL (e.g., https://abc123.ngrok.io)
```

### 3. Register MCP Server with Le Chat

In Le Chat, run:
```
/mcp register https://abc123.ngrok.io/mcp/manifest
```

**Expected Response**:
```
✅ MCP server registered: uDos Doc Manager
```

### 4. Verify Registration

Check registered MCP servers:
```
/mcp list
```

**Expected Response**:
```
Registered MCP Servers:
1. uDos Doc Manager (https://abc123.ngrok.io)
   - compile_docs
   - get_doc_report
```

### 5. Test MCP Command

Try running a command:
```
/mcp uDosDocManager.compile_docs {"source": "dev/**/*.md"}
```

### 6. Common Issues and Fixes

#### Issue: Server not running
**Fix**: Start the server (`node server.js`)

#### Issue: ngrok not exposing server
**Fix**:
- Check ngrok logs (`ngrok http 3000`)
- Verify local server is running (`curl http://localhost:3000/mcp/manifest`)

#### Issue: Invalid manifest
**Fix**:
- Validate manifest JSON (`jq . manifest.json`)
- Check required fields (name, version, commands)

#### Issue: Authentication failing
**Fix**:
- Add API key to server
- Register key with Le Chat (`/mcp auth abc123`)

#### Issue: CORS errors
**Fix**:
Add CORS headers to server:
```javascript
app.use((req, res, next) => {
  res.header("Access-Control-Allow-Origin", "*");
  res.header("Access-Control-Allow-Headers", "*");
  next();
});
```

### 9. Example MCP Server

```javascript
// server.js
const express = require('express');
const app = express();
app.use(express.json());

// CORS
app.use((req, res, next) => {
  res.header("Access-Control-Allow-Origin", "*");
  res.header("Access-Control-Allow-Headers", "*");
  next();
});

// Manifest
app.get('/mcp/manifest', (req, res) => {
  res.json({
    name: "uDos Doc Manager",
    version: "1.0",
    commands: [
      {
        name: "compile_docs",
        description: "Compile and compost documentation",
        parameters: {
          type: "object",
          properties: {
            source: { type: "string", default: "dev/**/*.md" }
          }
        }
      }
    ]
  });
});

// Command
app.post('/mcp/compile_docs', (req, res) => {
  console.log("Command received:", req.body);
  res.json({
    success: true,
    message: "Compiled docs successfully"
  });
});

app.listen(3000, () => console.log('MCP server running on port 3000'));
```
