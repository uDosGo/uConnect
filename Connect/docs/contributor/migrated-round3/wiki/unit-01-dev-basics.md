# Unit 01: Dev Control Plane Basics

## What This Repo Is

`uDOS-dev` is the **family coordination** repo: roadmaps, Cursor workspace
definitions, governance scripts, and cross-repo documentation layout. It does
not ship the runtime; it tells the family **how** coordinated changes land.

## What You Should Learn

By the end of this unit you should be able to:

- find the live status ledger and the canonical family roadmap
- explain why **`main`** is the default integration branch
- open the right doc for Cursor lanes vs repo-local `docs/`

## Practical How-To

1. Read the roadmap status and family roadmap.

```text
@dev/notes/roadmap/v2-roadmap-status.md
@dev/notes/roadmap/v2-family-roadmap.md
```

2. Read execution order after Workspaces 01–08.

```text
docs/workspace-08-exit-evidence.md
docs/cursor-execution.md
```

3. Skim documentation layout (three surfaces: `docs/`, `@dev/`, `wiki/`).

```text
docs/family-documentation-layout.md
```
