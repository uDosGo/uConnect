# uDos Ecosystem - Current Structure (April 2024)

## 🌐 High-Level Overview

The uDos ecosystem has been completely restructured into a modular, extensible platform
with clear separation of concerns between core platform, tools, and development framework.

## 🔧 Key Components

### 1. uDosGo (Core Platform)
- **Purpose**: Core business logic, skill execution, MCP server
- **Language**: Rust
- **Skills**: Rust skills (performance) + Vibe skills (flexibility)
- **Integration**: References external tools via symlinks

### 2. OkAgentDigital (Tools)
- **Purpose**: Standalone applications that integrate with uDosGo
- **Tools**: ThinUI, Hivemind, Re3Engine, PublishLane
- **Integration**: MCP sockets for communication
- **Development**: Independent lifecycle and versioning

### 3. DevStudio (Development)
- **Purpose**: Development tools and global configuration
- **Skills**: Development-focused skills (bash-doctor, build-guide)
- **Configuration**: Global settings for all components

### 4. User Skills (Personal)
- **Purpose**: Personal productivity and automation
- **Location**: `~/Vault/.vibe/skills/`
- **Examples**: note-taker, vault-tidy, data-indexer

## 🔄 Communication Flow

```
ThinUI → MCP → Orchestrator → uCode1 → Skills
          ← ←           ← ←
```

All components communicate via **MCP (Message Control Protocol)** using Unix sockets:
- Core: `~/Code/Vault/.uds/mcp.sock`
- Re3Engine: `~/.uds/re3engine.sock`
- Orchestrator: `~/Code/Vault/.uds/orchestrator.sock`

## 🚀 Development Workflow

### Adding New Tools
1. Create repository in OkAgentDigital organization
2. Implement MCP interface for communication
3. Add tool-specific skills in `vibe-skills/`
4. Reference from uDosGo as external dependency
5. Document integration points

### Building Components
```bash
# Core platform
cd ~/Code/uDosGo
cargo build --release

# ThinUI
cd ~/Code/OkAgentDigital/ThinUI
npm run tauri build

# All tools
cd ~/Code/OkAgentDigital/Hivemind
cargo build --release
```

## 📦 Configuration Files

### Global Configuration
```yaml
# ~/Code/DevStudio/config.yaml
global:
  mcp_sockets:
    core: "~/Code/Vault/.uds/mcp.sock"
    re3engine: "~/.uds/re3engine.sock"
    orchestrator: "~/Code/Vault/.uds/orchestrator.sock"
```

### Project Configuration
```yaml
# ~/Code/uDosGo/udos-config.yaml
tools:
  thinui:
    type: external
    path: "../OkAgentDigital/ThinUI"
  hivemind:
    type: external
    path: "../OkAgentDigital/Hivemind"
```
