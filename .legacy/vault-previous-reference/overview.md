# Course Overview

The universal Sonic Screwdriver framework is the portable deployment and
hardware-bootstrap lane in the repo family.

This course teaches one stable mental model:

`plan -> inspect -> dry-run -> apply -> hand off`

By the end of the course, a student should be able to:

- explain Sonic's repo boundary
- inspect a deployment layout and manifest
- use dry-run before destructive execution
- understand rescue and release-gate workflows
- identify when work belongs to `uDOS` or `uHOME-server` instead of Sonic
