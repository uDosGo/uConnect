# Chatdown v4.2.0 — Product Specification

**Version:** 4.2.0  
**Status:** Locked (refinable)  
**Role:** Document‑centric chat IDE that produces live markdown alongside conversation, with AI providers including Apple Intelligence (via Syncdown)  
**Principle:** Document is primary; chat is the co‑pilot.  
**Layout:** Document panel (left) | Chat panel (right)  
**Price:** Free (loss leader – drives Syncdown sales)

---

## 1. Core Identity

Chatdown is a **document‑centric chat interface**. You write a markdown document with the help of an AI assistant. The conversation builds the document; the document is the product.

**Mental model:** You're not chatting *at* an AI – you're *building* a document *with* an AI.

**Not a vault.** Not a task executor. Not a system health manager.  
**Just a focused, free workspace where conversation becomes structured markdown.**

**Chatdown is free.** It showcases Apple Intelligence (via Syncdown) and drives Syncdown sales.

---

## 2. Non‑Negotiable Rules (v4.2.x)

| Rule | Description |
|------|-------------|
| **Document on the left, chat on the right** | The document is the primary focus. Chat is a tool. |
| **One session → one document** | No tabs, no multi‑page. Each session produces a single markdown artefact. |
| **Document is read‑only** (v4.2.x) | No direct editing. Two‑way sync comes in v4.3+. |
| **No vault read/write** | Chatdown never reads from or writes to the vault. Handoff is explicit. |
| **No frontmatter editing** | Hidden metadata exists only for export/handoff (v4.1+). |
| **No system health** | Cleaning, composting, deduplication – owned by Linkdown. |
| **Ephemeral by default** | No persistent session storage. On close, user prompted to export or discard. |
| **Free** | No charge. Ever. Upsell to Syncdown for Apple Intelligence. |

## 4. Live Document Generation

### 4.1 Source

The document is generated **on the fly** from the conversation array (user messages + assistant responses).

```typescript
type Message = {
  role: "user" | "assistant";
  content: string;
  timestamp: Date;
  provider?: string;  // which provider generated this response
};
```

### 4.2 Markdown Structure

| Message type | Markdown output |
|--------------|----------------|
| **User message** | `## User: {timestamp}\n\n{content}\n` |
| **Assistant response** | `{content}\n` (no extra heading) |

### 4.3 Provider Attribution in Document

Optional setting: add a small footer to assistant responses showing which provider was used.

```markdown
_Generated by Apple Intelligence ✨_
```

## 5. Provider Integration

### 5.1 Syncdown (Apple Intelligence) – Primary Upsell

**Detection:** Chatdown checks `http://127.0.0.1:9876/health` on launch and periodically.

**If Syncdown is running and licensed:**
- Provider shows as "Syncdown ✨ (Apple Intelligence)"
- Chat calls `apple_intelligence.chat` MCP tool via Syncdown's local MCP server
- Responses are fast, private, offline‑capable

**If Syncdown is not running:**
- Provider dropdown shows "Syncdown ✨ (Apple Intelligence) – Not available"
- Banner appears: *"Get Apple Intelligence – free, private, offline – with Syncdown. Start 14‑day trial →"*
- Link opens Syncdown purchase page (or Mac App Store)

**If Syncdown trial has expired:**
- Banner: *"Your Syncdown trial has expired. Subscribe to continue using Apple Intelligence in Chatdown."*
- Chatdown still works with other providers (OpenAI, Ollama, etc.)

### 5.2 OAuth Connections for API Keys

Chatdown supports **OAuth 2.0** for providers that offer it (OpenAI, Anthropic, Google), eliminating the need for users to manually copy API keys.

**Supported OAuth providers:**

| Provider | OAuth Status | Notes |
|----------|--------------|-------|
| **OpenAI** | ✅ Planned | Via OpenAI OAuth (if available) or API key fallback |
| **Anthropic** | ✅ Planned | Via Anthropic OAuth |
| **Google Gemini** | ✅ Planned | Via Google OAuth (using user's Google account) |
| **Local (Ollama/LM Studio)** | ❌ Not needed | No authentication |

**OAuth Flow:**

```
User clicks "Connect" next to OpenAI provider
         │
         ▼
Chatdown opens system browser to provider's OAuth page
         │
         ▼
User logs in and authorises Chatdown
         │
         ▼
Provider redirects to localhost callback (chatdown://oauth/callback)
         │
         ▼
Chatdown exchanges code for access token
         │
         ▼
Token stored securely in macOS Keychain
         │
         ▼
Chatdown uses token for API calls
```

**Security:**
- Tokens stored in **macOS Keychain** (not on disk)
- Never logged or transmitted except to provider
- User can revoke access from provider's dashboard
- Option to manually enter API key as fallback

**UI Elements:**
- Provider row in Settings: "OpenAI – [Connect] [Manual API key]"
- After connection: "OpenAI – Connected (user@example.com) [Disconnect]"

### 5.3 Manual API Key Fallback

For users who prefer manual entry or when OAuth is unavailable:
- Text field for API key
- "Test Connection" button
- Key stored in Keychain (same as OAuth token)

### 5.4 Other Providers

| Provider | Integration method |
|----------|-------------------|
| **Ollama** | Detect local server at `http://127.0.0.1:11434`, list available models |
| **LM Studio** | Detect local server at `http://127.0.0.1:1234` |
| **Custom OpenAI‑compatible** | User enters base URL + API key |

## 7. Handoff to Linkdown (Stub in v4.2.0)

In v4.2.0, the **"Send to Linkdown"** button is **enabled** (v4.1+ feature).

**Handoff data structure:**

```typescript
interface ChatdownHandoff {
  version: "v4";
  page: {
    title: string;           // from first user message or user‑supplied
    kind: "note" | "tasks" | "research" | "brief" | "story";
    markdown: string;        // current live document content
  };
  meta: {
    intent?: string;
    linkdownTarget?: string;
    provider?: string;       // which provider generated the content
  };
}
```

**Flow:**
1. User clicks "Send to Linkdown"
2. Dialog: choose destination (Notes, Tasks, Research, etc.)
3. Chatdown saves handoff object to temp location
4. Linkdown (if running) picks it up and imports
5. If Linkdown not running, Chatdown saves to `.md` file in a default location

- Auto‑detect local server (toggle)
- Manual URL override

### 9.3 Keychain Storage

All credentials (OAuth tokens, API keys) stored in **macOS Keychain**:
- Service: `com.chatdown.provider.<provider>`
- Account: user email or "manual"
- Access: only Chatdown can read

## 10. Performance & Limits

| Metric | Limit / Behaviour |
|--------|-------------------|
| Conversation length | Unlimited (memory only) – warn after 100 messages |
| Document size | No hard limit; recommend < 10,000 words for responsive rendering |
| Rendering | Throttle updates if user types very fast (debounce 200ms) |
| OAuth token refresh | Automatic when expired (if provider supports refresh tokens) |
