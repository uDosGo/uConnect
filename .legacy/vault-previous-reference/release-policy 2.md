# uDOS-docs Release Policy

## Purpose

Define how `uDOS-docs` handles milestone tags and changelog discipline.

## Default Policy

`uDOS-docs` remains a docs-first repo.

Default behavior:

- milestone tags are optional, not required for every merge
- `CHANGELOG.md` records notable public documentation changes
- releases remain notes-only

## Tagging Rule

Use a tag only when the docs surface has a meaningful milestone change, such as:

- a large public onboarding rework
- a major family architecture reframe
- a substantial docs publication milestone

## Artifact Rule

`uDOS-docs` does not attach build artifacts.

Release value comes from:

- changelog-backed notes
- published documentation changes
- stable family references
