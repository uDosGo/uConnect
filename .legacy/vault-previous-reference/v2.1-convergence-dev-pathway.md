with implementation-ready handoffs.

## Scope

This pathway covers:

- Wizard OK control-plane spec relabeling into v2.1
- uCODE and script-system decision integration
- Ubuntu workstation decision integration
- ThinUI scaffold wiring and first themed boot render
- full-operation `@dev` module with OK ASSIST MCP support
- full-family launchability, quickstarts, and integration checks
- archive and previous-version decommission readiness

## Boundary Guardrails

- Core owns semantics and runtime truth
- ThinUI owns local GUI runtime loop
- Themes own visual adapters and token maps
- Wizard owns managed network/provider orchestration
- Ubuntu lane owns workstation/base-image scope, not semantic contracts
